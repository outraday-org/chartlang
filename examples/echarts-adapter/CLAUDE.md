# examples/echarts-adapter/

Example adapter — **not published to npm**. Renders chartlang emissions to
[Apache ECharts](https://echarts.apache.org) native series + `grid`
sub-panes. Mirrors `examples/canvas2d-adapter/` in shape (library package +
test seam + capabilities-only conformance default export).

## Conventions

- `package.json` carries `"private": true` and the unscoped name
  `chartlang-example-echarts-adapter`. No `publishConfig`. Same six-file
  §22.4 surface generated by `pnpm scaffold` — do not hand-edit the
  template files; edit `scripts/scaffold.ts` and re-run.
- The intent is "copy from this folder when writing your own ECharts
  adapter".
- ECharts is consumed via the `echarts` package; the option/series types
  (`EChartsOption`, `LineSeriesOption`, …) import from the
  `echarts/types/dist/echarts` subpath (the package's `.` entry is an
  `export =` namespace, which `verbatimModuleSyntax` rejects for a named
  `import type`).

## Invariants

- **The adapter drives ECharts DECLARATIVELY.** `onEmissions` ingests the
  drain into accumulated state, rebuilds ONE authoritative `EChartsOption`
  via `buildOption(state)`, and applies it with
  `chart.setOption(option, { notMerge: true })`. There is no imperative
  per-series mutation — every frame is the full option tree. A behavioural
  change re-shapes that tree and re-pins `integration.test.ts`'s
  `hashOptionLog` constant.

- **`opts.echartsFactory` is the mandatory instance seam.** Production
  passes `() => echarts.init(container)`; tests pass `() => new
  MockECharts()`. Unlike canvas2d's `opts.ctx` (which has a `getContext`
  fallback), there is NO implicit fallback — ECharts needs a DOM container
  + sizing the adapter does not own, so the factory is required.

- **The default export is capabilities-only.** `DEFAULT_ADAPTER`
  (`src/defaultAdapter.ts`) is the headless conformance export: the
  conformance runner reads `capabilities` only, so `candles` /
  `onEmissions` / `dispose` are no-ops — no ECharts instance is spun up for
  an emission-contract test. Production uses `createEChartsAdapter`.

- **`ECHARTS_CAPABILITIES` declares the FULL surface, including drawings.**
  The full Phase-5 plot inventory + all 63 drawing kinds (62
  `allPhase3Drawings()` + `table`) are declared NOW even though the drawing
  renderer lands in Task 10. Declaring the whole set keeps the adapter
  interchangeable with canvas2d and exercises the entire conformance
  surface. The capabilities surface is byte-for-byte the same shape as
  `CANVAS2D_CAPABILITIES`.

- **Plot kinds map to NATIVE ECharts facilities** (per `buildOption`):
  candlestick / line (`step:'end'` for step-line) / area (`areaStyle`) /
  bar (histogram + horizontal-histogram) / stacked line-pair (filled-band)
  / `markLine` (horizontal-line) / `scatter` (shape/marker/character/arrow/
  label). Candle-state overrides (`candle-override` / `bar-override` /
  `bar-color`) become per-bar candlestick `itemStyle`; `bg-color` becomes
  the chart `backgroundColor`. None of these create a series — they are
  applied during `applyPlot` and `StoredSeries.style` is narrowed
  (`SeriesStyle`) to EXCLUDE them, keeping `buildOption`'s series switch
  exhaustive over exactly the series-producing kinds.

- **Sub-panes are ECharts `grid`s.** One `grid` + x/y axis pair per
  `paneOrder` key (overlay = grid 0), mirroring canvas2d's `paneOrder`.
  Series route to their pane's `xAxisIndex` / `yAxisIndex`. `gridIndexOf`
  is lockstep-guaranteed (`applyPlot` pushes every pane before storing a
  series/hline for it) — its `-1` fallback is `v8 ignore`d as defensive.

- **`filled-band` bounds ride the per-point STYLE, not the value.** Each
  band emission's `upper`/`lower` are captured on `SeriesPoint.style`
  (only for `filled-band`); other kinds leave `style` undefined. A `null`
  bound (or a slot that switched to filled-band mid-stream, leaving a
  style-less earlier point) renders as a `'-'` gap.

- **`null` / non-finite values → the ECharts `'-'` gap; `visible: false`
  omits-not-deletes.** A hidden slot's prior points stay in state so
  re-enabling re-renders them.

- **Drawings render through the ECharts `graphic` component, NOT a `ctx`
  sink (Task 10).** `applyDrawing` buffers `DrawingEmission`s in
  `state.drawings` (keyed by `handleId`, last-write-wins, `op:"remove"` drops
  the key); `buildOption`'s `buildGraphics` then decomposes each live drawing
  via the shared `decomposeDrawing(emission, viewport)` (from
  `@invinite-org/chartlang-adapter-kit`) → `DrawPrimitive[]` and maps each
  primitive with `primitiveToGraphic` (`src/primitiveToGraphic.ts`) into one
  ECharts `graphic` element, set on `option.graphic`. ECharts is declarative,
  so there is NO `paintPrimitive` call — the adapter never imports the
  `/canvas` sink. Open polyline → `polyline`; closed → `polygon`; `arc` →
  `arc`; `text` → `text`; `marker` → small `circle` / `polygon`.
  `StrokeStyle.dash` → `style.lineDash` (omitted when solid); `alpha` →
  `strokeOpacity`; `FillStyle` → `fill` + `fillOpacity`. A behavioural change
  to the graphic mapping re-pins `integration.test.ts`'s `hashOptionLog`.

- **`EChartsGraphicElement` is a NARROW union, not ECharts' loose graphic
  type.** ECharts' `GraphicComponentLooseOption` is `Dictionary<any>`-loose;
  `primitiveToGraphic` returns the narrow `EChartsGraphicElement` instead and
  `buildOption` relies on its structural assignability to
  `EChartsOption["graphic"]` at the single `graphic:` assignment (verified by
  `pnpm typecheck`, no `as`). Style keys (`lineDash` / `fill` /
  `strokeOpacity` / `fillOpacity` / text `backgroundColor`) are OMITTED, not
  set to a sentinel, when the IR carries no value, keeping the option tree —
  and its hash — minimal.

- **NaN anchors are SKIPPED, not painted (divergence from the ctx adapters).**
  ECharts logs a console warning for a `graphic` element with a non-finite
  coordinate. `buildGraphics` filters each primitive through
  `primitiveIsFinite` (exported from `src/primitiveToGraphic.ts`); a
  non-finite primitive (NaN time/price → NaN pixel, or an empty polyline) is
  dropped entirely. The canvas-family adapters instead paint a harmless no-op
  path — documented in the README + the reference page.

- **The drawing `Viewport` comes from ECharts' grid pixels.**
  `buildViewport(chart, bars)` (`src/viewport.ts`) samples two grid corners
  via `chart.convertToPixel({ gridIndex: 0 }, [time, price])` and hands them to
  the pure `computeViewport`; the bar-time x extent comes from the bar window
  (the x axis is a CATEGORY of bar times, so `decomposeDrawing` does the
  time→x projection). `EChartsSurface` carries an OPTIONAL `convertToPixel?`:
  the headless default and a not-yet-laid-out chart omit it, and
  `buildViewport` then returns a deterministic fallback viewport so
  `option.graphic` is always well-defined. `MockECharts.convertToPixel`
  applies the deterministic affine `mockValueToPixel` so the sampling path is
  exercised headlessly (the pure `computeViewport` is verified against the
  `convertToPixel` identity in `viewport.test.ts`).

- **The in-package conformance test drives the FULL default scenario suite.**
  `src/conformance.test.ts` calls `runConformanceSuite(default)` and asserts
  `failed === 0` against the capabilities-only default export. It carries a
  300 s per-test timeout — the 200+ scenarios compile + run through esbuild +
  the runtime, far past vitest's 5 s default under v8 coverage. This adds
  `@invinite-org/chartlang-conformance` as a `devDependencies` entry.

- **No cross-example src imports.** The shared geometry / mock surface is
  consumed ONLY through the public `@invinite-org/chartlang-adapter-kit`
  (+ `/canvas`) and `@invinite-org/chartlang-host-worker` boundaries —
  never another example's `src/`.

- **`hashOptionLog` is a LOCAL hasher mirroring the canvas
  `hashCallLog` canonicalisation approach.** The public
  `hashCallLog` (adapter-kit/canvas) is typed against the canvas
  `RecordedCall` union and cannot accept ECharts `EChartsOption` trees;
  `src/testing.ts`'s `hashOptionLog` applies the same approach
  (canonical sorted-key JSON + 4-dp finite-float rounding + SHA-256) to
  `RecordedOptionCall[]`. Test assertions inspect the option tree
  structurally; the hash is a secondary stable pin.

- **`MockECharts` lives at `src/testing.ts`, exposed via the `./testing`
  sub-path** (parallel to canvas2d's `MockCanvas2DContext`). It records
  `setOption` / `resize` / `convertToPixel` / `dispose` calls; its
  `convertToPixel` applies the deterministic `mockValueToPixel` affine map (a
  named export, so the viewport test can reproduce the exact pixels). Coverage
  applies — `src/testing.test.ts` walks every method.
