## Requirements

### 1. Spec — `docs/spec/emissions.md`

Add a `z` row to the `PlotEmission` field table and the
`DrawingEmission` table: type `number?`, "presentation-only render-order
key; omitted ⇒ `0`; higher on top; finite numbers only; adapters sort by
`(z, groupBand, declarationOrder)`." Cross-link the
`docs/spec/semantics.md` render-ordering section (Task 1).

### 2. Language doc — `docs/language/series-and-indexing.md` (or a new short doc)

Add a short **"Layering (z-order)"** subsection: explain default group
order, declaration-order tiebreak, and the `z` option with a code
snippet showing a drawing pulled beneath a plot via `z: -1`. If a
dedicated `docs/language/layering.md` fits the docs structure better,
create that and link it from the nav/index — match the existing docs
organization.

### 3. Dedicated example script

Create `examples/scripts/z-layering.chart.ts` — a small, real indicator
that visibly demonstrates `z`. Suggested content: a shaded band via
`draw.fillBetween` given `z: -1` so it renders **behind** an SMA/price
`plot`, plus a second `plot` at `z: 1` to sit on top — proving cross-band
control. Note `draw.fillBetween`'s real signature is
`fillBetween(edgeA: ReadonlyArray<WorldPoint>, edgeB: ReadonlyArray<WorldPoint>, opts?: FillBetweenStyle)`
— build the two edges from `bar.point(...)` per bar, mirroring the
existing `examples/scripts/fill-between-band.chart.ts`. Include the
standard 2-line MIT copyright/header the other example scripts use
(`// Copyright (c) 2026 Invinite. Licensed under the MIT License.` /
`// See the LICENSE file in the repo root for full license text.`).
Verify it compiles via the example build/typecheck.

### 4. Demo entry + generated docs page

Add a `DemoScript` to `DEMO_SCRIPTS` in
`apps/site/src/components/demo/scripts.ts`:

```ts
// Build the band the same way examples/scripts/fill-between-band.chart.ts
// does (accumulate edgeA / edgeB as WorldPoint arrays across bars), then
// pass { z: -1 } so the fill sits BEHIND the plots. Skeleton:
const Z_LAYERING = `// Copyright (c) 2026 Invinite. Licensed under the MIT License.
// See the LICENSE file in the repo root for full license text.

import { defineIndicator } from "@invinite-org/chartlang-core";
export default defineIndicator({
    name: "Z-Order Layering",
    apiVersion: 1,
    overlay: true,
    compute({ bar, ta, plot, draw }) {
        // … construct fastEdge / slowEdge exactly as fill-between-band.chart.ts does …
        draw.fillBetween(fastEdge, slowEdge, { z: -1 }); // behind the price
        plot(bar.close, { title: "Price" });
        plot(ta.sma(bar.close, 20), { title: "SMA on top", z: 1 });
    },
});
`;

// add to DEMO_SCRIPTS:
{
    id: "z-layering",
    label: "Z-Order Layering",
    description: "Use the z option to render a fill behind the price and an SMA on top.",
    source: Z_LAYERING,
},
```

Keep the demo source and `examples/scripts/z-layering.chart.ts`
**consistent** (same logic). Then run `pnpm examples:generate` to emit
`docs/examples/z-layering.md` and update `docs/examples/index.md`;
commit the generated output (the `examples:gate` byte-diffs it).

### 5. Author skill — `skills/chartlang-coding/SKILL.md`

Extend the Task-1 "Layering / draw order" note to document the `z`
option now that it ships: default `0`, higher on top, finite numbers,
negative drawing `z` to go beneath plots. Keep it concise and point to

### 6. Skills reference regen — `pnpm skills:generate`

**The `z` option does NOT surface in the generated reference.**
`scripts/generate-skills-reference.ts` reads `draw.*` JSDoc from
`packages/runtime/src/emit/draw` and emits only the constructor
**signature, description, anchors, `@since`, stability** — it does
**not** extract option-type fields, and it does **not** cover `plot()` /
`PlotOpts` at all. Adding `z?` to the core option types (Task 2) leaves
`skills/chartlang-coding/references/primitives.md` **byte-identical**, so
there is nothing to regenerate for `z`.

Therefore: teach `z` entirely through **`SKILL.md` prose** (Requirement
5). Running `pnpm skills:generate` here is optional and should produce
**no `z`-related diff**; if it produces any diff, that diff is unrelated
to this feature — investigate before committing. The `skills:gate` must
be green either way.

### 7. Pine porting note — `references/translating-from-pine.md`

Add a short entry: "Pine `explicit_plot_zorder` — chartlang orders by
declaration within a group by default, so this flag needs no
translation; for explicit control use the chartlang `z` option on
`plot()` / `draw.*`." (This is the note deferred from Task 7.)

### 8. Edge cases / invariants

- The example MUST compile and pass the example typecheck/build — a
  broken `DEMO_SCRIPTS` source fails the site build and the docs gen.
- Run `pnpm examples:generate` **and** `pnpm skills:generate` and commit
  outputs; forgetting either trips `examples:gate` / `skills:gate`.
- Root `README.md` ≤ 300 lines, package READMEs ≤ 100 lines — if you add
  a layering mention to any README, respect the gate.
