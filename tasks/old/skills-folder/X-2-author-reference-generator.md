# Task 2 ‚Äî Author primitive-reference generator + drift gate

> **Status: TODO**

## Goal

Add `scripts/generate-skills-reference.ts` ‚Äî a tooling script that
walks the `ta.*` and `draw.*` source JSDoc (the same JSDoc the docs
site and hover registry consume) and emits
`skills/chartlang-coding/references/primitives.md`. Wire two `pnpm`
scripts: `skills:generate` (write) and `skills:gate` (`--check`,
regenerate-and-byte-diff, fails on drift). Add `skills:gate` to
`pnpm check`. This replaces Task 1's placeholder with authoritative,
never-drifting content.

## Prerequisites

Task 1 (the `skills/chartlang-coding/references/primitives.md`
placeholder + its auto-header sentinel must exist).

## Current Behavior

- `skills/chartlang-coding/references/primitives.md` is a Task-1
  placeholder.
- No script generates it; no gate catches drift.
- `scripts/gen-hover-registry.ts` already walks `packages/core/src`
  JSDoc via the TS compiler API and emits a checked-in artifact with a
  `--check` gate ‚Äî the pattern to reuse.

## Desired Behavior

After this task:

- `pnpm skills:generate` writes
  `skills/chartlang-coding/references/primitives.md` from source JSDoc.
- `pnpm skills:gate` regenerates into memory and byte-diffs against the
  committed file, failing with an actionable message on drift (mirrors
  `pnpm hover:check` / `pnpm docs:gate`).
- `pnpm check` runs `skills:gate`.
- The committed `primitives.md` contains every `ta.*` and `draw.*`
  primitive with signature, `@formula`, `@warmup`, `@since`, stability,
  and a one-line description.

## Requirements

### 1. `scripts/generate-skills-reference.ts`

Model the structure on `scripts/gen-hover-registry.ts`:

```ts

import { readFile, writeFile } from "node:fs/promises";

import ts from "typescript";

    REPO_ROOT,
```

**JSDoc extraction ‚Äî reuse the two existing per-file parsers, do not
re-implement parsing:**

- **`ta.*`** ‚Äî deep-import `parsePrimitiveSource` from
  `packages/cli/src/commands/genDocs.ts` (already an `export` at line
  ~238; returns a `PrimitiveDocInput`). Use the source-not-`dist` deep
  import precedent `docs-gate.ts` sets. Walk `TA_SRC` with a top-level
  `readdir` (subdirectories like `lib/` are skipped automatically by
  not descending). Apply the **same skip list** `genDocs.ts` uses:
  basenames `index.ts`, `persistence.ts`, `registry.ts`,
  `sourceValue.ts`, and every `*.test.ts` / `*.bench.ts` /
  `*.bench.test.ts` / `*.golden.test.ts` / `*.property.test.ts`. Do
  **not** call `runGenDocs` ‚Äî it writes pages to disk and returns
  `{ written, skipped }` paths, not `PrimitiveDocInput[]`.
- **`draw.*`** ‚Äî deep-import `parseDrawingSource` from
  `packages/cli/src/commands/extractDrawingPages.ts` (already an
  `export` at line ~176; returns a `DrawingDocInput`). Walk `DRAW_SRC`
  recursively ‚Äî drawings are organised by bucket subfolder
  (`lines/`, `boxes/`, ‚Ä¶). Skip the same `*.test.ts` / `*.bench.ts`
  suffixes plus `index.ts`. Do **not** call `runGenDrawingDocs` (same
  write-to-disk reason as `runGenDocs`).
- Each primitive contributes: signature, summary (first paragraph),
  `@formula`, `@warmup`, `@anchors` (draw only), `@since`, and the
  `@stable` / `@experimental` / `@frozen` marker.
- Cross-check the `ta.*` id set against
  `packages/core/src/statefulPrimitives.ts` so a primitive present in
  the registry but missing JSDoc is a **hard error** (not a silent
  drop) ‚Äî same spirit as the docs gate.

> Both `parsePrimitiveSource` and `parseDrawingSource` are already
> public exports of their source files, so no new export ‚Äî and no
> changeset ‚Äî is required. If a future refactor makes either
> file-private, the docs gate's deep-import precedent (`docs-gate.ts`)
> still applies; surface it back via the CLI barrel and document in
> `packages/cli/CLAUDE.md` per the root `CLAUDE.md` rule.

‚Ä¶ (one block per `draw.*` primitive ‚Äî bucket subfolders are flattened

The signature shown is the **script-facing** form (no leading
`slotId` ‚Äî the compiler injects it). Mirror `genDocs.ts`'s convention:
annotate once at the top that the compiler injects `slotId`, then show
clean `ta.<id>(...)` signatures.

**`generateSkillsReference(opts)`** exported function with
`{ check?: boolean }`:

- `check: true` ‚Üí read committed file; if missing or byte-unequal,
  `throw new Error("skills/chartlang-coding/references/primitives.md is
  out of date. Run pnpm skills:generate and commit.")`. Mirror
  `gen-hover-registry.ts`'s check branch verbatim in shape.
- default ‚Üí `writeFile(OUTPUT, contents)`.

A `--check` argv flag at the bottom selects the mode, exactly like
`gen-hover-registry.ts`:

```ts
// Mirror `scripts/gen-hover-registry.ts`'s entry-guard shape exactly:
if (process.argv[1] === fileURLToPath(import.meta.url)) {
    const check = process.argv.includes("--check");
    generateSkillsReference({ check }).catch((err: unknown) => {
        const message = err instanceof Error ? err.message : String(err);
        console.error(message);
        process.exitCode = 1;
    });
}
```
