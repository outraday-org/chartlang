# Conformance scenarios + docs/skills for the candle feature

> **Status: TODO**

## Goal

Close the `plotcandle` / `plotbar` feature: add conformance scenarios
that exercise both styles end-to-end (compile → runtime emit → adapter
render), refresh the generated docs + skills reference, and update the
hand-authored `SKILL.md` so authors discover the two functions.

## Prerequisites

Tasks 6 (runtime emit) and 7 (adapter render). Tasks 2/3 must also be
landed if their conformance scenarios were not yet registered (this task
does a final suite sweep).

## Current Behavior

`packages/conformance/src/scenarios/plotKindCandleOverride.scenario.ts`
covers the *color-only* `candle-override`. There is no scenario for the
value-carrying `candle` / `ohlc-bar`. `SKILL.md` documents `bgcolor` /
`barcolor` but not `plotcandle` / `plotbar`. The generated
`skills/chartlang-coding/references/primitives.md` and
`docs/primitives/` are regenerated by `pnpm skills:generate` /
`pnpm docs:generate`.

## Requirements

### 1. Conformance scenarios (`packages/conformance/src/scenarios/`)

Add `plotKindCandle.scenario.ts` + `plotKindOhlcBar.scenario.ts`,
mirroring `plotKindCandleOverride.scenario.ts` (use the `plotKindSource`
helper). Inline sources:

```ts
// candle
'plotcandle(bar.open, bar.high, bar.low, bar.close, { bull: "#26a69a", bear: "#ef5350", doji: "#999999" });'
// ohlc-bar
'plotbar(bar.open, bar.high, bar.low, bar.close, { color: "#f59e0b" });'
```

Assertions (mirror the candle-override scenario): a pinned `plot-hash`
(computed after Task 7 lands the deterministic emit — run once, then
pin from the runner's "expected vs actual" failure message, the
established re-pin workflow), `diagnostic-code-absent:
unsupported-plot-kind`, `diagnostic-code-absent: malformed-emission`.
`intervalCount: 1`, `candleLimit: 5`. Note `plot-hash` covers
`{ bar, value }` tuples only — Task 6's `value: close ?? null` is what
makes these scenarios hashable.

Wire both the way `PLOT_KIND_CANDLE_OVERRIDE_SCENARIO` is wired in
`packages/conformance/src/scenarios/index.ts`: import (~line 132),
re-export (~line 385), append to `ALL_SCENARIOS` (~line 574). There is
**no scenario-count gate** to bump: `phase2Inventory.ts` is the
hand-written inventory of *Phase-2 `ta.*` primitives* (not scenarios)
and these additions do not belong there — the registry cardinality
gates in `phase2Coverage.test.ts` were already updated by Tasks 2/3/5.
