# `chartlang add-adapter` installer + adapters registry + bundle generator

> **Status: TODO**

## Goal

Let a user drop a ready-to-use library adapter into their repo with one
command: `npx @invinite-org/chartlang-cli add-adapter <library>`. This
task builds the shared **adapters registry** (single source of truth), a
**bundle generator** that bakes each `examples/<adapter>/` folder into the
CLI as offline, version-pinned templates, and the `add-adapter` subcommand
(with `--list`, target dir, package-name + workspace-dep rewriting, and a
"next steps" printout). The same registry feeds the docs gallery in Task 15.

## Prerequisites

- Tasks 4, 6, 8, 10, 12 — all five example adapters exist and are
  conformance-green (canvas2d, lightweight-charts, uplot, echarts, konva).

## Current Behavior

- `@invinite-org/chartlang-cli` (public, bin `chartlang`, `parseArgs`
  dispatcher in `packages/cli/src/index.ts`) exposes `compile`,
  `scaffold-adapter` (emits 8 blank starter files from string templates in
  `src/adapterTemplate/templates.ts`), `docs`, `pine-convert`.
- Examples are **private** (`"private": true`) — not installable from npm.
  Today users either run `scaffold-adapter` (blank starter) or hand-copy
  `examples/canvas2d-adapter/` from GitHub
  (`https://github.com/outraday-org/chartlang/tree/main/examples/<adapter>`).
- No folder-copy/degit mechanism; no machine-readable adapter list.

## Desired Behavior

`chartlang add-adapter konva ./my-chart` writes a complete, working
konva adapter package into `./my-chart`, with `workspace:^` deps rewritten
to the published `@invinite-org/chartlang-*` versions matching the CLI
release, and prints how to install + run conformance.
`chartlang add-adapter --list` prints the comparison matrix.

## Requirements

### 1. Adapters registry (SSOT) — `scripts/adapters/registry.ts`

A typed, hand-maintained list — the one place new adapters are declared:

```ts
export type AdapterRegistryEntry = {
    readonly id: string;            // "konva" — the add-adapter arg
    readonly exampleDir: string;    // "examples/konva-adapter"
    readonly displayName: string;   // "Konva"
    readonly library: string;       // npm package "konva"
    readonly libraryRange: string;  // "^9"
    readonly license: string;       // "MIT"
    readonly renderTech: string;    // "Scene-graph (Canvas)"
    readonly strategy: "ctx" | "nodes" | "graphic" | "native-ctx"; // integration style
    readonly fullSurface: true;     // all 63 drawings + all plot kinds
    readonly approxBundleKb: number;// hand-maintained, deterministic for the gate
    readonly bestFor: string;       // one-line "choose this when…"
};
export const ADAPTERS: ReadonlyArray<AdapterRegistryEntry> = [ /* 5 entries */ ];
```

GitHub folder/README/conformance links are **derived** from `exampleDir` +
the canonical repo `https://github.com/outraday-org/chartlang` (a helper
`githubFolder(entry)`), not stored per-entry.

### 2. Bundle generator — `scripts/gen-adapters.ts` (`pnpm adapters:generate`)

For each registry entry, read `examples/<dir>/` and emit an offline
template bundle into the CLI source:

- Collect `src/**/*.ts` (include unit tests; **omit** `integration.test.ts`
  / `conformance.test.ts` only if they depend on workspace-internal
  fixtures the user won't have — document the inclusion list), plus
  `README.md`, `tsconfig.json`, `package.json`, `.gitignore`.
- **Rewrite `package.json`:** every `"@invinite-org/chartlang-*":
  "workspace:^"` → the published version read from that package's current
  `package.json` `version` (a `resolveWorkspaceVersions()` helper). Replace
  the `name` with a `__PKG_NAME__` placeholder the command substitutes.
  Keep `"private": true` (it's the user's app by default).
- Emit `packages/cli/src/generated/adapters/<id>.ts` exporting
  `{ id, files: Record<string, string> }` (path → contents) — the same
  "templates as string literals" shape as `adapterTemplate/templates.ts`,
  so the bundle is offline + deterministic.
- Emit `packages/cli/src/generated/adapters/index.ts` re-exporting all
  bundles + a `BUNDLED_ADAPTERS` array, and a generated `registry.ts`
  mirror (the metadata the CLI needs at runtime — id/library/license/etc.)
  so the CLI never imports from `scripts/`.
- Prepend a `// GENERATED by pnpm adapters:generate — DO NOT EDIT` header
  to every emitted file.

### 3. Gate — `pnpm adapters:gate`

Re-run the generator into a temp location and **byte-diff** against the
committed `packages/cli/src/generated/adapters/**`; fail on drift (mirrors
`docs:gate` / `examples:gate`). Wire into the repo's gate aggregation and
note it in `scripts/CLAUDE.md`. Task 15 extends this gate to also cover the
generated docs gallery.

### 4. `add-adapter` command — `packages/cli/src/commands/addAdapter.ts`

`chartlang add-adapter [library] [target-dir] [--list] [--name <pkg>] [--pm <npm|pnpm|yarn|bun>] [--force]`

- `--list` (or no args, non-TTY) → print the comparison matrix from
  `BUNDLED_ADAPTERS` (id, library, license, render tech, bundle size,
  best-for) + the exact `add-adapter <id>` command per row.
- `add-adapter <id> [dir]` → resolve the bundle; default `dir` = `./<id>-adapter`;
  refuse a non-empty dir unless `--force`; write every `files` entry
  (creating subdirs); substitute `__PKG_NAME__` with `--name` (default the
  dir basename); print next steps: `cd <dir>`, the detected/`--pm` install
  command, and `<pm> exec chartlang …` / how to run `runConformanceSuite`.
- Unknown `<id>` → error listing valid ids; exit code 1.
- Optional TTY niceness: if no arg **and** `process.stdin.isTTY`, prompt a
  selection via `node:readline/promises` (zero new deps); keep a
  non-interactive code path for tests (inject the chosen id).

Zero new runtime dependencies — use `node:fs/promises`, `node:path`,
`node:util.parseArgs`, `node:readline/promises` only.

### 5. Dispatcher + help — `packages/cli/src/index.ts`

Add the `case "add-adapter": await runAddAdapter(rest); return;` arm and a
help line. Document the relationship: `scaffold-adapter` = blank starter;
`add-adapter` = a complete library adapter you can run immediately.

### 6. Tests (co-located, 100% coverage)

- `commands/addAdapter.test.ts` — for each of the 5 ids: write into a temp
  dir, assert the file set + that `package.json` has no `workspace:` deps
  and the substituted name; `--list` output; unknown-id error;
  non-empty-dir refusal + `--force`; pm detection branches.
- `gen-adapters.test.ts` — generator output is stable (re-run = identical);
  `workspace:^` fully rewritten; `__PKG_NAME__` placeholder present in the
  bundle. This doubles as the `adapters:gate` contract.

### Edge cases

- Workspace dep rewrite must pin to the **exact** versions of the
  publishing release — derive from each package's `version` at generate
  time so a CLI version always installs matching peers.
- An example gaining/removing a file must change the committed bundle →
  `adapters:gate` red until regenerated (the intended drift guard).
- Windows path separators in the `files` map keys → normalise to `/` and
  re-join with `path` on write.
- `bin`-invoked via `pnpm dlx` must work offline (no network) — the bundle
  is baked in.
