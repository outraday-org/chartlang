# Task 14 — `add-adapter` CLI installer + registry + bundle generator

## Decisions (resolved from task text, no blocking ambiguity)

- **Bundle inclusion list** (per `examples/<dir>/`):
  - `src/**/*.ts` EXCEPT `integration.test.ts` and `conformance.test.ts`
    (both import workspace-internal host-worker test internals / pin a
    hashed call-log + drive the conformance suite as a CI fixture — the
    user can't reproduce these in a fresh copy). `__fixtures__/**` is
    included where present (canvas2d) — it is part of the unit tests.
  - `README.md`, `tsconfig.json`, `package.json` — copied/rewritten.
  - `.gitignore` — **synthesized** by the generator (no example ships one;
    the scaffold-adapter command's `node_modules/\ndist/\ncoverage/\n` is
    the canonical content). The user's fresh repo needs one.
  - `CHANGELOG.md` is NOT bundled (none exist; the `files` array reference
    is harmless for a private package).
- **package.json rewrite**: every `@invinite-org/chartlang-*` dep value of
  `workspace:^` → published version read from that package's current
  `package.json#version` (resolveWorkspaceVersions). `name` → `__PKG_NAME__`.
  Keep `private:true`. Drop nothing else.
- **approxBundleKb**: hand-maintained per registry entry (sum of bundled
  text KB, rounded) — deterministic, used only for the `--list` matrix.
- **Generated coverage**: add `src/generated/**` to cli vitest coverage
  `exclude` (machine-generated, same spirit as barrel/types exclusion).

## Gate contract

`gen-adapters.test.ts` re-runs the generator and byte-compares each emitted
file against the committed `packages/cli/src/generated/adapters/**` — the
same assertion `pnpm adapters:gate` makes. Determinism (sorted keys, fixed
header) doubles as the drift guard.
