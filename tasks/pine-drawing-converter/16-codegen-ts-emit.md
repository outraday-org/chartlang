# Task 16 — Codegen: chartlang TS emitter

> **Status: TODO**

## Goal

Walk the fully-populated `ScriptScaffold` IR (from Tasks 8–15) and
emit a deterministic, well-formatted chartlang `.chart.ts` source
string. The output must compile cleanly through
`@invinite-org/chartlang-compiler` and pass every chartlang gate
(forbidden-constructs analyzer, slot injection, JSDoc, README). This
is the converter's "back end" — every prior task contributes IR, this
task is what the user sees.

## Prerequisites

Tasks 8–15 (full IR coverage).

## Current Behavior

The `ScriptScaffold` IR is populated but no string emitter exists.
`convert(source)` returns `output: null` because the codegen layer is
the last to land.

## Desired Behavior

A package-internal `emit(scaffold: ScriptScaffold): string` API in
`src/codegen/emit.ts` produces a chartlang `.chart.ts` source string
of the shape:

```ts
// Auto-generated by @invinite-org/chartlang-pine-converter.
// Source: <pine-file-name> (if known)
// Do not edit directly — re-run the converter.

import { defineIndicator, ta, plot, hline, alert, draw, input, state, request, type DrawingHandle } from "@invinite-org/chartlang-core";

// ── converter helpers ──────────────────────────────────────────
const __BAR_INTERVAL_MS = <number>;
let __barCount = 0;
const __bar_index = (): number => __barCount;

type __HandleSlot<K extends string> = { current(): DrawingHandle | null; set(h: DrawingHandle | null): void };
function useDrawingHandleSlot<K extends string>(): __HandleSlot<K> { /* ... */ }

type __HandleRing<K extends string> = { /* ... */ };
function useDrawingHandleRing<K extends string>(cap: number): __HandleRing<K> { /* ... */ }
// ────────────────────────────────────────────────────────────────

// per-handle slot allocations
const __lvl_handle = useDrawingHandleSlot<"line">();
const __dash_handle = useDrawingHandleSlot<"table">();
const __pivots_ring = useDrawingHandleRing<"line">(50);

export default defineIndicator({
    name: "<from scaffold.name>",
    apiVersion: 1,                            // REQUIRED by compiler — hard-coded to 1 in v1
    overlay: <scaffold.overlay ?? true>,
    inputs: {
        <input declarations from scaffold.inputs>
    },
    maxDrawings: {
        <if any maxDrawings set>
    },
    compute({ bar, ta, plot, hline, alert, draw, inputs, request, state, barstate }) {
        // Note: the emitter destructures ONLY the ComputeContext fields the
        // body actually uses — `barstate`/`state` are nearly universal for
        // converted Pine scripts (every `var` slot needs `state`, every
        // `barstate.islast` gate needs `barstate`), so they're shown here
        // alongside the common surface. `syminfo`, `timeframe`, `runtime`,
        // `signal` are added on demand by the import-minimization walk in
        // §3 below.
        if (barstate.isnew) __barCount++;
        // <interleaved statements from scaffold.computeBody>
    },
});
```

The emitter is purely templating — no semantic decisions, no transforms.

## Requirements

### 1. Emitter architecture (`src/codegen/`)

- `emit.ts` — top-level entry; orchestrates the sections.
- `emitImports.ts` — chartlang core import list (minimized to only
  what's used).
- `emitHelpers.ts` — the converter helper preamble
  (`useDrawingHandleSlot`, `useDrawingHandleRing`, `__bar_index`,
  `__BAR_INTERVAL_MS`).
- `emitInputs.ts` — input object literal.
- `emitMaxDrawings.ts` — `maxDrawings` object literal (omit if all
  zero).
- `emitCompute.ts` — the `compute({...}) { ... }` body, statement by
  statement from `scaffold.computeBody`.
- `emitExpression.ts` — expression → string (reuses Task 7's
  `exprEmit` helper).
- `format.ts` — final formatting pass: indentation (4 spaces),
  semicolons, double-quoted string literals, no trailing whitespace,
  no `var`, prefer `const` — enough to satisfy Biome
  (`pnpm lint`) when the output lands on disk via `--out`. The
  emitted source must pass the workspace's Biome lint rules; the
  formatter encodes them at synthesis time so downstream consumers
  don't need a separate format pass. Test fixture: run
  `pnpm format` (workspace alias for `biome format --write .`) on
  every `*.expected.chart.ts` in Task 19's golden corpus and confirm
  diff is empty (the emitted strings are already canonical). The
  legacy `--apply-unsafe` flag was renamed in Biome v2; use
  `npx biome check --write --unsafe <path>` if a one-shot autofix is
  needed against a single file, or just rely on the converter's
  built-in formatter.

### 2. Determinism

Same scaffold → byte-identical output. Achieved by:

- Iteration over `Map`s uses insertion order (which mirrors source
  order, set by Task 5's semantic analyzer).
- No `Date`, `Math.random`, or wall-clock access.
- Identifier suffixes (`__lvl_handle_0`, `__lvl_handle_1`, …) are
  derived from the source span hash + a deterministic counter.

A test in `emit.property.test.ts` asserts `emit(s) === emit(s)`.

### 3. Import minimization

Walk the IR once to determine which chartlang surfaces are referenced:
`ta`, `plot`, `hline`, `alert`, `draw`, `input`, `state`,
`request`, `DrawingHandle`. Emit only those in the import list. The
`defineIndicator` / `defineDrawing` import is always present.

### 4. State and handle helper emission

For each `StateSlotIR`:

- `state.float(initial)` → emit `const <name> = state.float(<initial>);`
- `state.int(...)`, `state.bool(...)`, `state.string(...)` → same.
- `state.tick.float(...)` → emit `const <name> = state.tick.float(<initial>);`
- `HandleSlotIR` / `useDrawingHandleSlot<K>()` → emit at module top via the helper
  preamble.
- `HandleRingIR` / `useDrawingHandleRing<K>(cap)` → same.

### 5. Compute-body emission

Statement IR types (defined collectively by Tasks 10–15):

- `RawTsStatement` — already-stringified TS (most common; transforms
  produce these directly).
- `IfStatement` — recursive emit of branches.
- `ForStatement` — literal-bounded for.
- `SwitchStatement` — block.
- `BlockStatement` — `{ ... }`.
- `DrawCallStatement` — `draw.<kind>(args, opts)`.
- `HandleUpdateStatement` — `handle.update({...})`.
- `HandleRemoveStatement` — `handle.remove(); slot.set(null);`.

The emitter dispatches on the IR statement kind. Every statement has
a `sourceSpan` for diagnostics.

### 6. Comment preservation

Pine line comments map to chartlang `//` comments inline, preserving
relative position. Block comments don't exist in Pine. The `//@version=6`
directive is replaced by the auto-generated header comment.

### 7. Compiler round-trip verification

The emit pipeline ends with a call to
`@invinite-org/chartlang-compiler`'s `compile(output, { apiVersion: 1,
sourcePath: "<pine-name>.chart.ts" })`. **`compile` is async** —
returns `Promise<CompiledScript>`. If compilation fails (forbidden
constructs, type errors, JSDoc gate), emit `codegen-output-invalid`
error with the compiler's diagnostics attached. This is the
"the output had better compile" gate — without it the converter could
emit code that fails the chartlang gates.

Because `compile` is async, the round-trip verification step happens
in `convertFile` (async, defined in Task 18) rather than in `convert`
(sync). The sync `convert(source)` path emits the string but does
**not** round-trip; users who want round-trip assurance call
`convertFile` (or call `compile` themselves). Task 19's golden tests
drive `convertFile` for the round-trip property. This keeps the v1
public surface clean: sync emit; opt-in async verification.

`@invinite-org/chartlang-compiler` is declared as a `dependencies`
entry with `"workspace:*"` (matching every other intra-workspace
dependency — peerDependencies are reserved for the consumer-facing
editor package, not used internally).

### 8. The `convert(source)` public surface wiring

This task is where `convert` finally returns `{ output: ..., manifest:
..., diagnostics: ... }`:

```ts
// in src/index.ts (replacing the throw from Task 1)
// convert remains SYNC; round-trip compile-verify is opt-in via convertFile (Task 18, async).
export function convert(source: string, opts?: ConvertOpts): ConvertResult {
    const lexResult = lex(source);
    if (anyFatal(lexResult.diagnostics)) return { output: null, manifest: null, diagnostics: lexResult.diagnostics };
    const parseResult = parseStatements(lexResult.tokens);
    if (anyFatal(parseResult.diagnostics)) return { output: null, manifest: null, diagnostics: parseResult.diagnostics };
    const analysis = analyze(parseResult.script);
    const scaffold = createScaffold();
    transformDeclaration(parseResult.script.declaration, analysis, scaffold, diagnostics);
    transformInputs(analysis, scaffold, diagnostics);
    classifyAndTransformDrawings(analysis, scaffold, diagnostics);  // dispatch to campA/B/C
    transformTables(analysis, scaffold, diagnostics);
    transformPolylineLinefill(analysis, scaffold, diagnostics);
    transformOther(analysis, scaffold, diagnostics);
    const output = emit(scaffold);   // pure string emit, no compile dep
    return { output, manifest: scaffoldToManifest(scaffold), diagnostics: diagnostics.frozen() };
}
```

### 9. Manifest population

`scaffoldToManifest(scaffold)` produces the `ConvertManifest` defined
in Task 1: `kind`, `name`, `inputs` (names), `drawingKindsUsed` (set
of chartlang kinds), `requiresBarInterval` (true iff any
`bar-index-future` coordinate was resolved).

### 10. Diagnostic codes (added this task)

- `codegen-output-invalid` (error) — emitted output fails to compile
  through chartlang-compiler.
