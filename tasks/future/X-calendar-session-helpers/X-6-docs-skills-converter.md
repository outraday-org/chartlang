## Current Behavior

- Primitive doc pages are **auto-generated** ("AUTO-GENERATED by chartlang docs
  ‚Äî do not edit by hand") from core JSDoc; `docs/primitives/timeframe.md` is the
  template shape (Signature / Example / See also). The generator + sidebar are
  driven by `scripts/gen-examples-docs.ts` and `docs/.vitepress/config.ts`.
- The chartlang-coding skill reference
  (`skills/chartlang-coding/references/primitives.md`) is re-emitted by
  `pnpm skills:generate` (`scripts/generate-skills-reference.ts`); `skills:gate`
  fails CI if it is stale (repo-root `CLAUDE.md`).
- Examples live in `examples/scripts/*.chart.ts` (`examples/CLAUDE.md`); the
  demo list is `apps/site/src/components/demo/scripts.ts` (`DEMO_SCRIPTS`).
- Pine‚Üíchartlang identifier remap: `BUILTIN_IDENTIFIER_MAP`
  (`packages/pine-converter/src/mapping/builtinIdentifiers.ts:29`) maps `time` ‚Üí
  `bar.time`, OHLCV, `bar_index`. `dayofweek` / `session` are known builtins
  (`semantic/builtins.ts:151-152`) but have no lowering. Input-kind mapping is
  `INPUT_MAP` (`packages/pine-converter/src/mapping/inputs.ts:28`); there is no
  `input.session` row.

### 2. Determinism contract note (hand-authored prose page or section)

Add a short **"Time & timezones"** section to a hand-authored docs page (e.g.
`docs/language/` ‚Äî a new `time-and-sessions.md`, or extend `series-and-indexing`
/ an existing semantics page; follow `docs/CLAUDE.md` for placement). It must
state:

- `bar.time` is UTC ms epoch; calendar fields come from `time.*` (authors must
  not use `Date`/`Intl` ‚Äî they are banned).
- **v1 is UTC + fixed-offset only.** A real IANA zone with DST resolves to UTC
  and raises a `tz-dst-unsupported` warning. This is deliberate ‚Äî there is no
  self-contained tz database and `Intl` output varies by host ICU/tz-data
  version (the same reason `ta.sessionVolumeProfile` uses UTC-only math).
- `dayofweek` is Pine's `1=Sun..7=Sat`.
- Link to the Deferred section (exchange-tz/DST follow-up).

Cross-link it from the determinism / versioning spec if one enumerates the
reproducibility guarantees.

### 5. Pine-converter mapping

- **`dayofweek`**: add a lowering so the identifier / `dayofweek` (and
  `dayofweek(time, tz)` call form) maps to `time.dayofweek(bar.time)`. The
  cleanest seam is the expression emitter ‚Äî either extend
  `BUILTIN_IDENTIFIER_MAP` (`mapping/builtinIdentifiers.ts`) for the bare
  identifier and/or a call-form mapping where `ta.*`-style passthrough lives.
  `dayofweek` is already a known builtin (`semantic/builtins.ts:151`), so this
  is the missing emission, not a new recognition.
- **`time()`**: the bare `time` identifier already remaps to `bar.time`
  (`builtinIdentifiers.ts:38`). Confirm `time()` (no-arg call) and
  `time(timeframe.period)` degrade sensibly; a `time(session_spec)` membership
  form maps to `session.isOpen(bar.time, <spec>)`. Scope to the common forms;
  push an existing "unmapped" diagnostic for the exotic ones rather than
  emitting wrong code.
- **`input.session`**: add `["input.session", { pine: "input.session",
  chartlang: "input.session" }]` to `INPUT_MAP`
  (`mapping/inputs.ts:28`). The default-value path is a string literal, already
  handled by `literalDefault` via the generic `resolveDefault` branch ‚Äî add a
  unit test that `input.session("0930-1600")` lowers to
  `input.session("0930-1600")`.
- Add converter unit tests (mirror `ta-passthrough.test.ts` / `inputs.test.ts`)
  and, if the round-trip corpus is the right home, a golden fixture line ‚Äî but
  keep new Pine fixtures **compile-clean** (`conformance/CLAUDE.md` round-trip
  rule).
- Regenerate the converter support docs (`scripts/gen-converter-docs.ts` ‚Üí
  `docs/converter/supported.md`) so `dayofweek` / `input.session` move from
  unsupported to supported.
