## Overview

chartlang already ships excellent *human* docs (`docs/`,
auto-generated `docs/primitives/ta/<id>.md`) and per-folder `CLAUDE.md`
files that orient an AI agent working *inside this repo*. What it does
**not** have is a portable, installable knowledge pack an *external*
LLM can load to (a) write `.chart.ts` indicators in someone else's
editor, or (b) wire the compiler/runtime/adapter stack into a product.

This folder adds two skills under a repo-root `skills/` directory, in
the [skills.sh](https://www.skills.sh) / Anthropic Agent-Skill format
(`skills/<name>/SKILL.md` with `name` + `description` YAML
frontmatter). They install into any agent via
`npx skills add outraday-org/chartlang` (which discovers the `skills/`
dir and symlinks each skill into `.claude/skills/`, `.cursor/`, etc.)
or by manual copy.

- **`chartlang-author`** — for an end user wiring AI into their code
  editor to help write chartlang scripts. Teaches the language model
  the `defineIndicator`/`compute` contract, the import + destructure
  rule, forbidden constructs, inputs, and the `ta.*` / `draw.*`
  surface. Its primitive reference is **generated** from the same
  JSDoc the docs site uses, so it never drifts.
- **`chartlang-integrator`** — for a developer integrating chartlang
  into their project. Teaches the three integration paths — embed in
  your chart, write a chart adapter, run server-side alerts — with
  reference snippets pointing at the in-repo examples.

## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| **Anthropic SKILL.md format under repo-root `skills/`** | The format skills.sh's `npx skills add` discovers (`skills/<name>/SKILL.md`, `name`+`description` frontmatter). Also uploadable to Claude.ai and droppable into `.cursor/rules/`. Public + discoverable in the OSS repo. |
| **Author primitive reference is generated, not hand-written** | 90 `ta.*` + 61 `draw.*` primitives drift constantly. Reuse the `gen-hover-registry.ts` JSDoc-walk + `--check` gate pattern so a stale reference fails CI, exactly like `hover:check` / `docs:gate`. |
| **SKILL.md bodies are hand-written; only `primitives.md` is generated** | The narrative contract (import+destructure rule, forbidden constructs, four script kinds) is stable prose, not derivable from JSDoc. Generating only the primitive table keeps the gate surface small. |
| **Integrator skill ships reference snippets only — no scaffold generator** | Lowest maintenance, ships fastest. The skill points at `examples/canvas2d-adapter/` + `examples/react-demo/` rather than maintaining a parallel starter generator. |
| **Install via skills.sh `npx skills add`, not a new CLI subcommand** | No `@invinite-org/chartlang-cli` change → no published-package version bump, no changeset. skills.sh already resolves `owner/repo` → `skills/` dir. |
| **Root `CLAUDE.md` is terse: rules + index only** | Per-folder `CLAUDE.md` files already carry the deep orientation. The root file's only new job is the maintenance contract ("update folder CLAUDE.md + skills/ on behavior changes") plus a one-line index. Keeping it short avoids a second source of truth that itself rots. |
| **`skills/` excluded from coverage / JSDoc / README gates** | Markdown + a tooling script, not published TS source. The generator script follows the `scripts/` convention (MIT header, no JSDoc gate, no `dist/`). |

adds the second, independent skill.
