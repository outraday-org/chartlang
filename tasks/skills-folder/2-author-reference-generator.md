# Task 2 — Author primitive-reference generator + drift gate

> **Status: TODO**

## Goal

Add `scripts/generate-skills-reference.ts` — a tooling script that
walks the `ta.*` and `draw.*` source JSDoc (the same JSDoc the docs
site and hover registry consume) and emits
`skills/chartlang-author/references/primitives.md`. Wire two `pnpm`
scripts: `skills:generate` (write) and `skills:gate` (`--check`,
regenerate-and-byte-diff, fails on drift). Add `skills:gate` to
`pnpm check`. This replaces Task 1's placeholder with authoritative,
never-drifting content.

## Prerequisites

Task 1 (the `skills/chartlang-author/references/primitives.md`
placeholder + its auto-header sentinel must exist).

## Current Behavior

- `skills/chartlang-author/references/primitives.md` is a Task-1
  placeholder.
- No script generates it; no gate catches drift.
- `scripts/gen-hover-registry.ts` already walks `packages/core/src`
  JSDoc via the TS compiler API and emits a checked-in artifact with a
  `--check` gate — the pattern to reuse.

## Desired Behavior

After this task:

- `pnpm skills:generate` writes
  `skills/chartlang-author/references/primitives.md` from source JSDoc.
- `pnpm skills:gate` regenerates into memory and byte-diffs against the
  committed file, failing with an actionable message on drift (mirrors
  `pnpm hover:check` / `pnpm docs:gate`).
- `pnpm check` runs `skills:gate`.
- The committed `primitives.md` contains every `ta.*` and `draw.*`
  primitive with signature, `@formula`, `@warmup`, `@since`, stability,
  and a one-line description.

## Requirements

### 1. `scripts/generate-skills-reference.ts`

Two-line MIT header (per `scripts/CLAUDE.md` gate-script convention).
Model the structure on `scripts/gen-hover-registry.ts`:

```ts
// Copyright (c) 2026 Invinite. Licensed under the MIT License.
// See the LICENSE file in the repo root for full license text.

import { readFile, writeFile } from "node:fs/promises";
import { dirname, join, relative, resolve } from "node:path";
import { fileURLToPath } from "node:url";
import ts from "typescript";

const REPO_ROOT = resolve(dirname(fileURLToPath(import.meta.url)), "..");
const TA_SRC = join(REPO_ROOT, "packages/runtime/src/ta");
// draw.* JSDoc lives in core declarations, extracted by the CLI's
// extractDrawingPages.ts (NOT a runtime/src/draw dir — there is none).
const DRAW_SRC = join(REPO_ROOT, "packages/core/src/draw");
const OUTPUT = join(
    REPO_ROOT,
    "skills/chartlang-author/references/primitives.md",
);

export const AUTO_HEADER =
    "<!-- AUTO-GENERATED by pnpm skills:generate — do not edit by hand -->";
```

**JSDoc extraction — reuse the two existing extractors, do not
re-implement parsing:**

- **`ta.*`** — reuse the walk in `packages/cli/src/commands/genDocs.ts`.
  Enumerate `TA_SRC`, applying the **same skip list** `genDocs.ts`
  uses: exclude `index.ts`, `registry.ts`, `sourceValue.ts`, every
  `*.test.ts` / `*.bench.ts` / `*.bench.test.ts` / `*.golden.test.ts` /
  `*.property.test.ts`, and do **not** descend into `lib/`. Import the
  extraction function (`runGenDocs` / its underlying per-file
  `PrimitiveDocInput` collector) from `genDocs.ts` directly — that is
  the source-not-`dist` import precedent `docs-gate.ts` already sets.
  Render from the returned `PrimitiveDocInput[]` rather than re-parsing.
- **`draw.*`** — reuse `packages/cli/src/commands/extractDrawingPages.ts`
  (the extractor behind `docs/primitives/draw/<id>.md`). It already
  reads draw JSDoc from `packages/core/src/draw`. Consume its output
  shape; do not fork a parallel draw parser.
- Each primitive contributes: signature, summary (first paragraph),
  `@formula`, `@warmup`, `@anchors` (draw only), `@since`, and the
  `@stable` / `@experimental` / `@frozen` marker.
- Cross-check the `ta.*` id set against
  `packages/core/src/statefulPrimitives.ts` so a primitive present in
  the registry but missing JSDoc is a **hard error** (not a silent
  drop) — same spirit as the docs gate.

> If reusing `genDocs.ts` / `extractDrawingPages.ts` requires exporting
> a function that is currently file-private, export it (and add it to
> the package's JSDoc surface per the gate) rather than copy-pasting
> the parser. Document the new export in `packages/cli/CLAUDE.md` in
> the same PR (root `CLAUDE.md` rule).

**Rendering.** Emit Markdown:

```markdown
<!-- AUTO-GENERATED by pnpm skills:generate — do not edit by hand -->

# chartlang primitive reference

Generated from source JSDoc. Authoritative — do not hand-edit. Run
`pnpm skills:generate` after changing a `ta.*` / `draw.*` primitive.

## ta.*

### ta.ema

​```ts
ta.ema(source: Series<number>, length: number): Series<number>
​```

Exponential moving average. **Formula:** `…`. **Warmup:** `length − 1`
bars (NaN until filled). **Since:** 0.1 · stable.

… (one block per primitive, alphabetical within namespace) …

## draw.*

… (only if DRAW_SRC exists this phase) …
```

The signature shown is the **script-facing** form (no leading
`slotId` — the compiler injects it). Mirror `genDocs.ts`'s convention:
annotate once at the top that the compiler injects `slotId`, then show
clean `ta.<id>(...)` signatures.

**`generateSkillsReference(opts)`** exported function with
`{ check?: boolean }`:

- `check: true` → read committed file; if missing or byte-unequal,
  `throw new Error("skills/chartlang-author/references/primitives.md is
  out of date. Run pnpm skills:generate and commit.")`. Mirror
  `gen-hover-registry.ts`'s check branch verbatim in shape.
- default → `writeFile(OUTPUT, contents)`.

A `--check` argv flag at the bottom selects the mode, exactly like
`gen-hover-registry.ts`:

```ts
if (import.meta.url === pathToFileURL(process.argv[1]).href) {
    const check = process.argv.includes("--check");
    generateSkillsReference({ check }).catch((err) => {
        console.error(err instanceof Error ? err.message : err);
        process.exit(1);
    });
}
```
